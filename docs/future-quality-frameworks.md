# Future Quality Frameworks

The current implementation uses custom Python logic for lightweight, dependency-free validation. As the project grows, richer data quality frameworks may be evaluated.

## Potential Frameworks

Tools such as Great Expectations, Soda, Deequ, or others may be considered based on evolving requirements. No framework has been selected as the final choice yet.

The project remains open to future integrations while maintaining its current local-first approach.
